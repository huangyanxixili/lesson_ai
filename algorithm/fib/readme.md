# 斐波那契数列
从 0 和 1开始，后续每一项都等于前两项之和的整数序列
f(n) = f(n-1) + f(n-2)

- 递归
    复杂问题分解为多个小问题（类似的小问题）自顶向下
缺点：
    靠函数入栈实现 调用栈 栈内存爆栈
    重复计算 

大任务拆分为小任务时 满足树形结构 -> 一定可以用递归
    在拆分过程中，也可以观察出递归公式

## 优化重复计算
- hashMap 记录已经计算过的结果
- 如果计算过，直接从缓存中去除，不用入栈那么多函数

## IIFE 立即执行函数
    Immediately Invoked Function Expression
   - 非常适合创建闭包
   - 1、立即执行
   - 2、返回一个函数，形成闭包
   - (function() {}) ()
