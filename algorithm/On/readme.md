- 如何评价算法的好坏？
   - 时间复杂度 执行时间
   - 空间复杂度 占用内存空间

- 时间复杂度怎么计算？
   - 计算执行次数 T(n)，反映代码的执行时间 
   - 算法的时间复杂度，不是展现逻辑代码，而是随着输入规模的增大，反映算法的执行时间增长趋势。
     抓主要矛盾（忽略常数，取最高次数）
     O(n)

## 时间复杂度
- O(1)     常数时间
- O(logn)  对数时间
- O(n)     线性时间
- O(nlogn) 线性对数时间
- O(n^2)   二次时间
- O(n^3)   三次时间
- O(2^n)   指数时间
- O(n!)    阶乘时间

## 空间复杂度
一个算法在**运行过程**中临时占据存储空间大小的量度
