# react 通信组件

- 跨层级通信
   - 父子组件通信思想尝试解决跨层级通信的问题（props 传递）--> 被动接收
     一路传递数据，中间组件也不需要，被迫传递，及其麻烦
     规矩： 父组件（外层组件）负责持有和改变数据
       - 大于父子层级（2层），传递的路径就太长了
       - 数据在查找的上下文里面，在最外层时，提供给任何里面的任意层次组件随便使用  
   - 要消费数据/数据状态的组件 拥有自己查找数据的能力 --> 主动查找

   - Context
       - 总结：
           1. 创建上下文 (Create Context) ：使用 createContext 创建一个 Context 对象，它充当跨组件通信的 载体 。
           2. 提供数据 (Provide) ：在顶层使用该 Context 对象的 <Provider> 组件包裹子树，并通过 value 属性 广播 共享状态。
           3. 消费数据 (Consume) ：在任意深度的后代组件中，使用 useContext Hook 订阅 该 Context 对象，从而直接获取并响应最新的数据变化。
        一句话解释： 
         先实例化一个Context对象作为数据容器，再用Context对象内置的 Provider组件来提供数据，最后在需要使用数据的组件中使用 useContext 接收输出的数据容器
