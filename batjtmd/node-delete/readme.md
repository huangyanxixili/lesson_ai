# 删除链表节点

## dummy 节点
- 节点迭代
- 删除节点
- dummy 节点（哨兵节点）虚拟的节点
    是一个人为添加的节点，不存储真实数据的假节点，通常放在列表的最前面（有时候在尾部），目的是简化链表操作的边界条件
    用空间换逻辑统一

## 链表的反转
    通过 dummy 节点 + 头插法
    dummy 充当了 反转后的头节点
    想清楚每轮的关系
  - 头插法分为三部：
    1. 保存下一节点             `const next = cur.next`
    2. 让当前节点指向已经反转的头 `cur.next = dummy.next`
    3. 成为新的反转头           `dummy.next = cur`

## 快慢指针 / 多指针
快慢指针是两个一前一后的指针，两个指针往同一个方向走，只是一个快、一个慢
- 判断链表是否有环
  快指针每次走两步，慢指针每次走一步
  如果链表有环，快指针最终会追上慢指针
  否则，快指针会先达到链表的末尾（null），说明链表无环+

## 删除链表的**倒数**第 N 个节点
dummy 节点 + 快慢指针

- 从 head 遍历一遍 --> length
- 倒数第　N 个节点 = 正数第 length - N + 1 个节点

- 让快指针先移动 N 步（使两个指针距离为 N）
- 慢指针再同时移动
- 当快指针达到末尾时，慢指针就指向倒数第N个节点的前一个
